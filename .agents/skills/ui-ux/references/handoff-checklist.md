# UI UX Handoff Checklist

Use this checklist before implementation sign-off.

## Product clarity
- [ ] Screen has one primary goal.
- [ ] Primary CTA is obvious within first viewport.
- [ ] Empty/loading/error/success states are defined.

## Interaction quality
- [ ] Primary flow can be completed without confusion.
- [ ] Microcopy is short, specific, and action-oriented.
- [ ] Dangerous actions include confirmation or undo.

## Visual consistency
- [ ] Uses a defined token system (color/type/space).
- [ ] Components are reused consistently.
- [ ] Visual hierarchy is obvious at a glance.

## Responsive behavior
- [ ] Works at 360px, 768px, and 1280px+ widths.
- [ ] No clipped text or overlapping UI.
- [ ] Touch targets are comfortable on mobile.

## Accessibility basics
- [ ] Keyboard navigation works for primary actions.
- [ ] Focus indicators are visible.
- [ ] Text contrast is readable on all major surfaces.
- [ ] Labels and helper text are descriptive.

## Performance and implementation
- [ ] Motion is purposeful and lightweight.
- [ ] Heavy visual effects are limited and optional.
- [ ] Design decisions are documented for engineering handoff.
