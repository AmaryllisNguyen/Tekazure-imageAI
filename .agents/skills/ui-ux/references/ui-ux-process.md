# UI UX Process

Use this process when asked to create or improve product UI/UX.

## 1) Frame the mission

Capture these five items before visual work:
- User: who is the primary user?
- Job: what are they trying to complete?
- Trigger: what brings them to this screen?
- Outcome: what success looks like for user and business?
- Constraint: brand, time, tech, legal, or data limits.

## 2) Define success criteria

Use concrete, testable targets:
- Task completion rate
- Time to first key action
- Error rate on primary form/action
- Drop-off at each step of the flow

## 3) Map the flow

Write the shortest realistic path:
- Entry
- Decision point
- Main action
- Confirmation
- Next action

Then define system states:
- Loading
- Empty
- Error
- Success

## 4) Shape the visual language

Lock these tokens first:
- Color roles: bg, surface, text, muted, primary, accent, danger, success.
- Type scale: 12, 14, 16, 20, 24, 32.
- Space scale: 4, 8, 12, 16, 24, 32, 48.
- Radius and shadow scale.

## 5) Build hierarchy

Each screen should include:
- One dominant title and one clear primary CTA.
- Grouped secondary actions.
- Predictable placement for status and feedback.

## 6) Validate before handoff

Minimum review:
- Mobile width 360px
- Tablet width 768px
- Desktop width 1280px
- Keyboard-only navigation
- Basic color contrast checks

## 7) Handoff output

Always provide:
- Final screen intent and flow summary
- Design token table
- Component inventory
- Interaction notes
- Edge-state behavior
